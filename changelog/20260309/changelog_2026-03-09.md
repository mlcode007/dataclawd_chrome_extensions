# 修改日报 — 2026-03-09

## 一、线上 URL 异常修复

**问题**：打包后线上运行时 API URL 拼接异常，导致获取任务和数据回传均失败。

**改动文件**：
- `js/content/isolate.js` — 修复回传接口 URL 拼接逻辑
- `js/panel/index.js` — 修复获取任务接口 URL 异常

## 二、数据回传重试机制

**问题**：数据回传超时或失败时直接丢失数据，无容错能力。

**改动文件**：
- `js/content/isolate.js` — 回传失败时自动重试最多 5 次，并在日志中记录重试次数；超时时间调整为 60 秒

## 三、云主机部署兼容性调研

**问题**：插件部署到云 Windows 主机后，关闭远程连接或分辨率变化会导致浏览器插件失效。

**改动文件**：
- 无代码改动，进行了技术调研（Service Worker 保活、浏览器最小化执行方案等）

## 涉及文件汇总

| 文件 | 改动点 |
|------|--------|
| `js/content/isolate.js` | URL 修复、回传重试机制、超时调整 |
| `js/panel/index.js` | 任务接口 URL 修复 |

---

## 总结

修复了打包后线上运行的 URL 拼接异常，恢复了任务获取和数据回传功能；新增回传失败自动重试机制（最多 5 次），显著提升了数据采集的可靠性；同时针对云主机部署场景进行了技术调研，为后续稳定运行方案提供了方向。
