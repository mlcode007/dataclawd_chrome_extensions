# 修改日报 — 2026-03-13

## 一、后台自动执行功能

**问题**：每次需要手动打开侧边栏并点击按钮才能启动采集任务，无法实现无人值守运行。

**改动文件**：
- `js/background/index.js` — 新增后台自动执行逻辑（+398行），支持浏览器启动时自动开始任务；增加任务状态管理和定时器防重复
- `js/panel/index.js` — 新增"后台执行"复选框，勾选后无需打开侧边栏即可自动执行；页面右上角显示关键词请求倒计时
- `js/content/isolate.js` — 配合后台执行优化数据传递
- `side_panel.html` — 增加复选框和倒计时 UI 元素

## 二、自动执行逻辑优化与任务关闭完善

**问题**：重复点击复选框导致多个定时器并行；关闭侧边栏后自动任务未正确停止；关闭任务时倒计时未立即结束。

**改动文件**：
- `js/background/index.js` — 修复定时器重复启动问题，完善任务关闭逻辑（关闭侧边栏时根据复选框状态决定是否停止任务）
- `js/panel/index.js` — 关闭自动任务后立即清除倒计时，不再等待当前周期结束

## 三、小红书自动登录与自动退出

**问题**：多账号切换需要手动登录退出，效率低下。

**改动文件**：
- `js/panel/index.js` — 新增自动登录流程（+411行）：导航到登录页 → 输入手机号 → 通过接码链接获取验证码 → 自动填入并提交；新增自动退出流程：点击页面退出按钮 → 清空 cookie；验证码重试最多 40 次
- `manifest.json` — 增加 cookies 权限支持退出时清空 cookie
- `side_panel.html` — 增加手机号、接码链接配置项和自动登录/退出按钮

## 四、Chrome 定时重启脚本

**问题**：云主机上 Chrome 长时间运行后内存增长，需要定期重启。

**改动文件**：
- `scripts/run_chrome_rednote_1min.bat` — Windows 批处理脚本，定时关闭并重启 Chrome 浏览器

## 涉及文件汇总

| 文件 | 改动点 |
|------|--------|
| `js/background/index.js` | 后台自动执行、定时器防重复、任务关闭完善 |
| `js/panel/index.js` | 后台执行复选框、倒计时展示、自动登录/退出流程 |
| `js/content/isolate.js` | 配合后台执行优化 |
| `side_panel.html` | 复选框、倒计时、登录配置 UI |
| `manifest.json` | cookies 权限 |
| `scripts/run_chrome_rednote_1min.bat` | Chrome 定时重启脚本 |

## 总结

实现了插件的无人值守运行能力——后台自动执行功能让浏览器启动即可自动采集，无需手动操作；自动登录/退出机制支持多账号自动切换，大幅降低了运维人力成本；同时修复了多个任务调度稳定性问题，并提供了 Chrome 定时重启脚本应对内存增长，整体提升了系统在云主机环境下的可靠性。
